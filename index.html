STIG WN11-CC-000305

Indexing of encrypted files must be turned off.

<#
.SYNOPSIS
    Remediates STIG WN11-CC-000305 - Disable indexing of encrypted files

.DESCRIPTION
    Disables Windows Search indexing of encrypted files by setting the
    appropriate Group Policy registry value.

.NOTES
    Run as Administrator
#>

Write-Host "Applying STIG WN11-CC-000305 remediation..." -ForegroundColor Cyan

# Registry path for Windows Search policy
$Path = "HKLM:\SOFTWARE\Policies\Microsoft\Windows\Windows Search"

# Create key if it doesn't exist
if (-not (Test-Path $Path)) {
    New-Item -Path $Path -Force | Out-Null
}

# Disable indexing of encrypted files
New-ItemProperty -Path $Path `
    -Name "AllowIndexingEncryptedStoresOrItems" `
    -PropertyType DWord `
    -Value 0 `
    -Force | Out-Null

# Restart Windows Search service to apply change
Write-Host "Restarting Windows Search service..." -ForegroundColor Yellow
Restart-Service -Name "WSearch" -Force -ErrorAction SilentlyContinue

# Validation
$result = Get-ItemProperty -Path $Path -Name "AllowIndexingEncryptedStoresOrItems" -ErrorAction SilentlyContinue

if ($result.AllowIndexingEncryptedStoresOrItems -eq 0) {
    Write-Host "STIG WN11-CC-000305 successfully remediated." -ForegroundColor Green
} else {
    Write-Host "STIG remediation FAILED." -ForegroundColor Red
}

Indexing of encrypted files may expose sensitive data. This setting prevents encrypted files from being indexed.

STIG WN11-CC-000252

Windows 11 must be configured to disable Windows Game Recording and Broadcasting.

<#
.SYNOPSIS
    Remediates STIG WN11-CC-000252 - Disable Windows Game Recording and Broadcasting

.DESCRIPTION
    Disables Xbox Game Bar background recording and game DVR features.
    Required for Windows 11 STIG compliance.

.NOTES
    Run as Administrator
#>

Write-Host "Applying STIG WN11-CC-000252 remediation..." -ForegroundColor Cyan

# Registry path for Game DVR / recording features
$Path = "HKLM:\SOFTWARE\Policies\Microsoft\Windows\GameDVR"

# Create key if it doesn't exist
if (-not (Test-Path $Path)) {
    New-Item -Path $Path -Force | Out-Null
}

# Disable Game DVR (recording/broadcasting)
New-ItemProperty -Path $Path `
    -Name "AllowGameDVR" `
    -PropertyType DWord `
    -Value 0 `
    -Force | Out-Null

# ALSO disable Xbox Game Bar background capture (user context policy)
$UserPath = "HKCU:\System\GameConfigStore"

if (-not (Test-Path $UserPath)) {
    New-Item -Path $UserPath -Force | Out-Null
}

New-ItemProperty -Path $UserPath `
    -Name "GameDVR_Enabled" `
    -PropertyType DWord `
    -Value 0 `
    -Force | Out-Null

# Validation
$check1 = Get-ItemProperty -Path $Path -Name "AllowGameDVR" -ErrorAction SilentlyContinue
$check2 = Get-ItemProperty -Path $UserPath -Name "GameDVR_Enabled" -ErrorAction SilentlyContinue

if ($check1.AllowGameDVR -eq 0 -and $check2.GameDVR_Enabled -eq 0) {
    Write-Host "STIG WN11-CC-000252 successfully remediated." -ForegroundColor Green
} else {
    Write-Host "STIG remediation FAILED or incomplete." -ForegroundColor Red
}

Windows Game Recording and Broadcasting is intended for use with games; however, it could potentially record screen shots of other applications and expose sensitive data. Disabling the feature will prevent this from occurring

STIG WN11-CC-000326

PowerShell script block logging must be enabled on Windows 11.

<#
.SYNOPSIS
    Remediates STIG WN11-CC-000326 - Enable PowerShell Script Block Logging

.DESCRIPTION
    Enables PowerShell Script Block Logging to ensure full visibility
    of executed PowerShell commands for auditing and security monitoring.

.NOTES
    Run as Administrator
#>

Write-Host "Applying STIG WN11-CC-000326 remediation..." -ForegroundColor Cyan

# Registry path for PowerShell Script Block Logging
$Path = "HKLM:\SOFTWARE\Policies\Microsoft\Windows\PowerShell\ScriptBlockLogging"

# Create key if it doesn't exist
if (-not (Test-Path $Path)) {
    New-Item -Path $Path -Force | Out-Null
}

# Enable Script Block Logging
New-ItemProperty -Path $Path `
    -Name "EnableScriptBlockLogging" `
    -PropertyType DWord `
    -Value 1 `
    -Force | Out-Null

# Optional but recommended: enable module logging as well (good for SIEM correlation)
$ModulePath = "HKLM:\SOFTWARE\Policies\Microsoft\Windows\PowerShell\ModuleLogging"
if (-not (Test-Path $ModulePath)) {
    New-Item -Path $ModulePath -Force | Out-Null
}

New-ItemProperty -Path $ModulePath `
    -Name "EnableModuleLogging" `
    -PropertyType DWord `
    -Value 1 `
    -Force | Out-Null

# Ensure module logging includes all modules
$ModuleSettingsPath = "$ModulePath\ModuleNames"
if (-not (Test-Path $ModuleSettingsPath)) {
    New-Item -Path $ModuleSettingsPath -Force | Out-Null
}

New-ItemProperty -Path $ModuleSettingsPath `
    -Name "*" `
    -PropertyType String `
    -Value "*" `
    -Force | Out-Null

# Validation
$check = Get-ItemProperty -Path $Path -Name "EnableScriptBlockLogging" -ErrorAction SilentlyContinue

if ($check.EnableScriptBlockLogging -eq 1) {
    Write-Host "STIG WN11-CC-000326 successfully remediated." -ForegroundColor Green
} else {
    Write-Host "STIG remediation FAILED." -ForegroundColor Red
}

Maintaining an audit trail of system activity logs can help identify configuration errors, troubleshoot service disruptions, and analyze compromises that have occurred, as well as detect attacks. Audit logs are necessary to provide a trail of evidence in case the system or network is compromised. Collecting this data is essential for analyzing the security of information assets and detecting signs of suspicious and unexpected behavior. Enabling PowerShell script block logging will record detailed information from the processing of PowerShell commands and scripts. This can provide additional detail when malware has run on a system.

STIG WN11-00-000160

Disables SMBv1 protocol.

<#
.SYNOPSIS
    Remediates STIG WN11-00-000160 by disabling SMBv1 on Windows 11 systems.

.DESCRIPTION
    The Server Message Block (SMB) v1 protocol is an outdated and insecure protocol.
    This script disables SMBv1 via Windows Optional Features and registry configuration.

.NOTES
    Author: YourName
    Date: 2026-05-03
    STIG ID: WN11-00-000160
#>

Write-Host "Checking SMBv1 status..." -ForegroundColor Cyan

# Disable SMBv1 via Windows Optional Feature (client component)
Try {
    Disable-WindowsOptionalFeature -Online -FeatureName "SMB1Protocol" -NoRestart -ErrorAction Stop
    Write-Host "SMBv1 Windows Feature disabled successfully." -ForegroundColor Green
}
Catch {
    Write-Host "SMBv1 feature may already be disabled or not present." -ForegroundColor Yellow
}

# Disable SMBv1 via registry (additional enforcement)
$registryPath = "HKLM:\SYSTEM\CurrentControlSet\Services\LanmanServer\Parameters"

If (!(Test-Path $registryPath)) {
    New-Item -Path $registryPath -Force | Out-Null
}

New-ItemProperty -Path $registryPath -Name "SMB1" -PropertyType DWORD -Value 0 -Force | Out-Null

Write-Host "Registry settings applied to disable SMBv1." -ForegroundColor Green

# Verify SMBv1 is disabled
Write-Host "`nVerifying SMBv1 state..." -ForegroundColor Cyan
Get-WindowsOptionalFeature -Online -FeatureName "SMB1Protocol"

Write-Host "`nRemediation complete. A reboot may be required for full effect." -ForegroundColor Green

SMBv1 is a legacy protocol that uses the MD5 algorithm as part of SMB. MD5 is known to be vulnerable to a number of attacks such as collision and preimage attacks as well as not being FIPS compliant. Disabling SMBv1 support may prevent access to file or print sharing resources with systems or devices that only support SMBv1. File shares and print services hosted on Windows Server 2003 are an example, however Windows Server 2003 is no longer a supported operating system. Some older Network Attached Storage (NAS) devices may only support SMBv1

STIG WN11-CC-000315

Windows Installer must not always install with elevated privileges

<#
.SYNOPSIS
    Remediates STIG WN11-CC-000315
    Windows Installer must not always install with elevated privileges

.DESCRIPTION
    The Windows Installer "AlwaysInstallElevated" policy allows non-admin users
    to install software with SYSTEM-level privileges. This is a CAT I (High)
    finding and must be disabled in both HKLM and HKCU hives.

.STIG
    ID:       WN11-CC-000315
    Severity: CAT I (High)
    Fix:      Set AlwaysInstallElevated to 0 in both registry hives

.NOTES
    Must be run as Administrator
#>

# ─────────────────────────────────────────
# Requires Administrator
# ─────────────────────────────────────────
If (-NOT ([Security.Principal.WindowsPrincipal] `
    [Security.Principal.WindowsIdentity]::GetCurrent()).IsInRole(`
    [Security.Principal.WindowsBuiltInRole] "Administrator")) {
    Write-Warning "This script must be run as Administrator. Exiting."
    Exit 1
}

# ─────────────────────────────────────────
# Registry Paths
# ─────────────────────────────────────────
$HKLMPath = "HKLM:\SOFTWARE\Policies\Microsoft\Windows\Installer"
$HKCUPath = "HKCU:\SOFTWARE\Policies\Microsoft\Windows\Installer"
$ValueName = "AlwaysInstallElevated"
$RequiredValue = 0

# ─────────────────────────────────────────
# Function: Check Current Value
# ─────────────────────────────────────────
Function Get-STIGStatus {
    param ($Path, $Name)

    If (Test-Path $Path) {
        $val = Get-ItemProperty -Path $Path -Name $Name -ErrorAction SilentlyContinue
        If ($null -ne $val.$Name) {
            Return $val.$Name
        }
    }
    Return "Not Set"
}

# ─────────────────────────────────────────
# Check Before Remediation
# ─────────────────────────────────────────
Write-Host "`n[*] Checking current state before remediation..." -ForegroundColor Cyan

$HKLMBefore = Get-STIGStatus -Path $HKLMPath -Name $ValueName
$HKCUBefore = Get-STIGStatus -Path $HKCUPath -Name $ValueName

Write-Host "  HKLM AlwaysInstallElevated : $HKLMBefore"
Write-Host "  HKCU AlwaysInstallElevated : $HKCUBefore"

# ─────────────────────────────────────────
# Remediate HKLM
# ─────────────────────────────────────────
Write-Host "`n[*] Remediating HKLM..." -ForegroundColor Yellow

Try {
    If (-Not (Test-Path $HKLMPath)) {
        New-Item -Path $HKLMPath -Force | Out-Null
        Write-Host "  [+] Created registry path: $HKLMPath"
    }
    Set-ItemProperty -Path $HKLMPath -Name $ValueName -Value $RequiredValue -Type DWord -Force
    Write-Host "  [+] HKLM\$ValueName set to $RequiredValue" -ForegroundColor Green
} Catch {
    Write-Error "  [-] Failed to remediate HKLM: $_"
}

# ─────────────────────────────────────────
# Remediate HKCU
# ─────────────────────────────────────────
Write-Host "`n[*] Remediating HKCU..." -ForegroundColor Yellow

Try {
    If (-Not (Test-Path $HKCUPath)) {
        New-Item -Path $HKCUPath -Force | Out-Null
        Write-Host "  [+] Created registry path: $HKCUPath"
    }
    Set-ItemProperty -Path $HKCUPath -Name $ValueName -Value $RequiredValue -Type DWord -Force
    Write-Host "  [+] HKCU\$ValueName set to $RequiredValue" -ForegroundColor Green
} Catch {
    Write-Error "  [-] Failed to remediate HKCU: $_"
}

# ─────────────────────────────────────────
# Verify After Remediation
# ─────────────────────────────────────────
Write-Host "`n[*] Verifying remediation..." -ForegroundColor Cyan

$HKLMAfter = Get-STIGStatus -Path $HKLMPath -Name $ValueName
$HKCUAfter = Get-STIGStatus -Path $HKCUPath -Name $ValueName

Write-Host "  HKLM AlwaysInstallElevated : $HKLMAfter"
Write-Host "  HKCU AlwaysInstallElevated : $HKCUAfter"

# ─────────────────────────────────────────
# Pass/Fail Result
# ─────────────────────────────────────────
Write-Host ""
If ($HKLMAfter -eq $RequiredValue -and $HKCUAfter -eq $RequiredValue) {
    Write-Host "  [PASS] WN11-CC-000315 remediation successful." -ForegroundColor Green
} Else {
    Write-Host "  [FAIL] WN11-CC-000315 remediation may not be complete. Review values above." -ForegroundColor Red
}

Write-Host "`n[*] Done. Run 'gpupdate /force' to apply Group Policy changes.`n" -ForegroundColor Cyan

Prevents silent privilege escalation.

STIG WN11-CC-000345

The Windows Remote Management (WinRM) service must not use Basic authentication.

<#
.SYNOPSIS
    Remediates STIG by disabling Basic Authentication for WinRM.

.DESCRIPTION
    The Windows Remote Management (WinRM) service must not use Basic authentication.
    This script disables Basic authentication for both WinRM Service and Client
    via registry policy settings and WinRM configuration.

.NOTES
    Author: YourName
    Date: 2026-05-03
#>

Write-Host "Disabling WinRM Basic Authentication..." -ForegroundColor Cyan

# -----------------------------
# Disable via Group Policy Registry (Service)
# -----------------------------
$serviceRegPath = "HKLM:\SOFTWARE\Policies\Microsoft\Windows\WinRM\Service"

If (!(Test-Path $serviceRegPath)) {
    New-Item -Path $serviceRegPath -Force | Out-Null
}

New-ItemProperty -Path $serviceRegPath `
    -Name "AllowBasic" `
    -PropertyType DWord `
    -Value 0 `
    -Force | Out-Null

Write-Host "WinRM Service Basic Authentication disabled via policy." -ForegroundColor Green

# -----------------------------
# Disable via Group Policy Registry (Client)
# -----------------------------
$clientRegPath = "HKLM:\SOFTWARE\Policies\Microsoft\Windows\WinRM\Client"

If (!(Test-Path $clientRegPath)) {
    New-Item -Path $clientRegPath -Force | Out-Null
}

New-ItemProperty -Path $clientRegPath `
    -Name "AllowBasic" `
    -PropertyType DWord `
    -Value 0 `
    -Force | Out-Null

Write-Host "WinRM Client Basic Authentication disabled via policy." -ForegroundColor Green

# -----------------------------
# Disable via WinRM configuration layer
# -----------------------------
Try {
    winrm set winrm/config/service/auth "@{Basic=`"false`"}" | Out-Null
    winrm set winrm/config/client/auth "@{Basic=`"false`"}" | Out-Null
    Write-Host "WinRM service and client auth updated successfully." -ForegroundColor Green
}
Catch {
    Write-Host "WinRM configuration update failed or already enforced by policy." -ForegroundColor Yellow
}

# -----------------------------
# Verification
# -----------------------------
Write-Host "`nCurrent WinRM Service Auth Settings:" -ForegroundColor Cyan
winrm get winrm/config/service/auth

Write-Host "`nCurrent WinRM Client Auth Settings:" -ForegroundColor Cyan
winrm get winrm/config/client/auth

Write-Host "`nRemediation complete. A restart of WinRM service may be required." -ForegroundColor Green

Basic authentication uses plain text passwords that could be used to compromise a system.

STIG WN11-CC-000285

Disables Guest account.

<#
.SYNOPSIS
    Remediates STIG WN11-CC-000285 - Require Secure RPC Communications for RDP

.DESCRIPTION
    Configures Remote Desktop Session Host to require secure RPC communications,
    ensuring encrypted and authenticated RPC traffic for RDP sessions.

.NOTES
    Run as Administrator
#>

Write-Host "Applying STIG WN11-CC-000285 remediation..." -ForegroundColor Cyan

# Registry path for RDP Session Host security settings
$Path = "HKLM:\SOFTWARE\Policies\Microsoft\Windows NT\Terminal Services"

# Create key if it doesn't exist
if (-not (Test-Path $Path)) {
    New-Item -Path $Path -Force | Out-Null
}

# Require secure RPC communication
# 1 = Require secure RPC (STIG compliant)
New-ItemProperty -Path $Path `
    -Name "fEncryptRPCTraffic" `
    -PropertyType DWord `
    -Value 1 `
    -Force | Out-Null

# Alternative/related enforcement depending on policy baseline
New-ItemProperty -Path $Path `
    -Name "fPromptForPassword" `
    -PropertyType DWord `
    -Value 1 `
    -Force | Out-Null

# Optional: enforce TLS for RDP connections
New-ItemProperty -Path $Path `
    -Name "SecurityLayer" `
    -PropertyType DWord `
    -Value 2 `
    -Force | Out-Null

# Optional: force Network Level Authentication (strongly recommended with STIG)
New-ItemProperty -Path $Path `
    -Name "UserAuthentication" `
    -PropertyType DWord `
    -Value 1 `
    -Force | Out-Null

# Validation
$result = Get-ItemProperty -Path $Path -Name "fEncryptRPCTraffic" -ErrorAction SilentlyContinue

if ($result.fEncryptRPCTraffic -eq 1) {
    Write-Host "STIG WN11-CC-000285 successfully remediated." -ForegroundColor Green
} else {
    Write-Host "STIG remediation FAILED." -ForegroundColor Red
}

Allowing unsecure RPC communication exposes the system to man in the middle attacks and data disclosure attacks. A man in the middle attack occurs when an intruder captures packets between a client and server and modifies them before allowing the packets to be exchanged. Usually the attacker will modify the information in the packets in an attempt to cause either the client or server to reveal sensitive information.
