WN11-CC-000305 Indexing of encrypted files must be turned off Medium

Indexing of encrypted files may expose sensitive data. This setting prevents encrypted files from being indexed by Windows Search.

PowerShell 
<#
.SYNOPSIS  Remediates STIG WN11-CC-000305 — Disable indexing of encrypted files
.NOTES     Run as Administrator
#>

Write-Host "Applying STIG WN11-CC-000305..." -ForegroundColor Cyan

$Path = "HKLM:\SOFTWARE\Policies\Microsoft\Windows\Windows Search"

if (-not (Test-Path $Path)) { New-Item -Path $Path -Force | Out-Null }

New-ItemProperty -Path $Path `
    -Name "AllowIndexingEncryptedStoresOrItems" `
    -PropertyType DWord -Value 0 -Force | Out-Null

Restart-Service -Name "WSearch" -Force -ErrorAction SilentlyContinue

$result = Get-ItemProperty -Path $Path -Name "AllowIndexingEncryptedStoresOrItems" -ErrorAction SilentlyContinue

if ($result.AllowIndexingEncryptedStoresOrItems -eq 0) {
    Write-Host "STIG WN11-CC-000305 successfully remediated." -ForegroundColor Green
} else {
    Write-Host "Remediation FAILED." -ForegroundColor Red
}
WN11-CC-000252 Windows Game Recording and Broadcasting must be disabled Medium

Game Recording could capture and expose sensitive data from other applications. Disabling it prevents inadvertent screen capture.

PowerShell 
<#
.SYNOPSIS  Remediates STIG WN11-CC-000252 — Disable Windows Game Recording and Broadcasting
.NOTES     Run as Administrator
#>

Write-Host "Applying STIG WN11-CC-000252..." -ForegroundColor Cyan

$Path = "HKLM:\SOFTWARE\Policies\Microsoft\Windows\GameDVR"
if (-not (Test-Path $Path)) { New-Item -Path $Path -Force | Out-Null }

New-ItemProperty -Path $Path -Name "AllowGameDVR" -PropertyType DWord -Value 0 -Force | Out-Null

$UserPath = "HKCU:\System\GameConfigStore"
if (-not (Test-Path $UserPath)) { New-Item -Path $UserPath -Force | Out-Null }

New-ItemProperty -Path $UserPath -Name "GameDVR_Enabled" -PropertyType DWord -Value 0 -Force | Out-Null

$check1 = Get-ItemProperty -Path $Path     -Name "AllowGameDVR"    -ErrorAction SilentlyContinue
$check2 = Get-ItemProperty -Path $UserPath -Name "GameDVR_Enabled" -ErrorAction SilentlyContinue

if ($check1.AllowGameDVR -eq 0 -and $check2.GameDVR_Enabled -eq 0) {
    Write-Host "STIG WN11-CC-000252 successfully remediated." -ForegroundColor Green
} else {
    Write-Host "Remediation FAILED or incomplete." -ForegroundColor Red
}
WN11-CC-000326 PowerShell script block logging must be enabled Medium

Script block logging records detailed PowerShell execution data, enabling audit trail analysis and malware detection. Module logging is also enabled for SIEM correlation.

PowerShell 
<#
.SYNOPSIS  Remediates STIG WN11-CC-000326 — Enable PowerShell Script Block Logging
.NOTES     Run as Administrator
#>

Write-Host "Applying STIG WN11-CC-000326..." -ForegroundColor Cyan

$Path = "HKLM:\SOFTWARE\Policies\Microsoft\Windows\PowerShell\ScriptBlockLogging"
if (-not (Test-Path $Path)) { New-Item -Path $Path -Force | Out-Null }

New-ItemProperty -Path $Path -Name "EnableScriptBlockLogging" -PropertyType DWord -Value 1 -Force | Out-Null

# Enable module logging (recommended for SIEM correlation)
$ModulePath = "HKLM:\SOFTWARE\Policies\Microsoft\Windows\PowerShell\ModuleLogging"
if (-not (Test-Path $ModulePath)) { New-Item -Path $ModulePath -Force | Out-Null }

New-ItemProperty -Path $ModulePath -Name "EnableModuleLogging" -PropertyType DWord -Value 1 -Force | Out-Null

$ModuleNamesPath = "$ModulePath\ModuleNames"
if (-not (Test-Path $ModuleNamesPath)) { New-Item -Path $ModuleNamesPath -Force | Out-Null }

New-ItemProperty -Path $ModuleNamesPath -Name "*" -PropertyType String -Value "*" -Force | Out-Null

$check = Get-ItemProperty -Path $Path -Name "EnableScriptBlockLogging" -ErrorAction SilentlyContinue

if ($check.EnableScriptBlockLogging -eq 1) {
    Write-Host "STIG WN11-CC-000326 successfully remediated." -ForegroundColor Green
} else {
    Write-Host "Remediation FAILED." -ForegroundColor Red
}
WN11-00-000160 SMBv1 protocol must be disabled High

SMBv1 uses the MD5 algorithm and is vulnerable to collision and preimage attacks. It is not FIPS compliant. Disabling it may affect legacy NAS devices or Windows Server 2003 shares (no longer supported). A reboot may be required.

PowerShell 
<#
.SYNOPSIS  Remediates STIG WN11-00-000160 — Disable SMBv1
.NOTES     Date: 2026-05-03 | STIG ID: WN11-00-000160
#>

Write-Host "Checking SMBv1 status..." -ForegroundColor Cyan

Try {
    Disable-WindowsOptionalFeature -Online -FeatureName "SMB1Protocol" -NoRestart -ErrorAction Stop
    Write-Host "SMBv1 Windows Feature disabled." -ForegroundColor Green
} Catch {
    Write-Host "SMBv1 feature may already be disabled or absent." -ForegroundColor Yellow
}

$regPath = "HKLM:\SYSTEM\CurrentControlSet\Services\LanmanServer\Parameters"
if (!(Test-Path $regPath)) { New-Item -Path $regPath -Force | Out-Null }

New-ItemProperty -Path $regPath -Name "SMB1" -PropertyType DWORD -Value 0 -Force | Out-Null
Write-Host "Registry enforcement applied." -ForegroundColor Green

Get-WindowsOptionalFeature -Online -FeatureName "SMB1Protocol"
Write-Host "Reboot may be required for full effect." -ForegroundColor Green
WN11-CC-000315 Windows Installer must not always install with elevated privileges High — CAT I

AlwaysInstallElevated allows non-admin users to install software with SYSTEM-level privileges — a critical privilege escalation vector. Must be disabled in both HKLM and HKCU hives.

PowerShell 
<#
.SYNOPSIS  Remediates STIG WN11-CC-000315 — Disable AlwaysInstallElevated
.NOTES     Severity: CAT I (High) | Must run as Administrator
#>

#Requires -RunAsAdministrator

$HKLMPath  = "HKLM:\SOFTWARE\Policies\Microsoft\Windows\Installer"
$HKCUPath  = "HKCU:\SOFTWARE\Policies\Microsoft\Windows\Installer"
$ValueName = "AlwaysInstallElevated"

foreach ($Path in @($HKLMPath, $HKCUPath)) {
    if (-Not (Test-Path $Path)) { New-Item -Path $Path -Force | Out-Null }
    Set-ItemProperty -Path $Path -Name $ValueName -Value 0 -Type DWord -Force
    Write-Host "Set $Path\$ValueName = 0" -ForegroundColor Green
}

$hklmVal = (Get-ItemProperty -Path $HKLMPath -Name $ValueName -ErrorAction SilentlyContinue).$ValueName
$hkcuVal = (Get-ItemProperty -Path $HKCUPath -Name $ValueName -ErrorAction SilentlyContinue).$ValueName

if ($hklmVal -eq 0 -and $hkcuVal -eq 0) {
    Write-Host "[PASS] WN11-CC-000315 remediation successful." -ForegroundColor Green
} else {
    Write-Host "[FAIL] Review values above." -ForegroundColor Red
}

Write-Host "Run 'gpupdate /force' to apply Group Policy changes." -ForegroundColor Cyan
WN11-CC-000345 WinRM service must not use Basic authentication High

Basic authentication transmits credentials in plain text, making them trivially interceptable. This script disables Basic auth for both the WinRM service and client via Group Policy registry and WinRM configuration layers.

PowerShell 
<#
.SYNOPSIS  Remediates STIG WN11-CC-000345 — Disable WinRM Basic Authentication
.NOTES     Date: 2026-05-03
#>

Write-Host "Disabling WinRM Basic Authentication..." -ForegroundColor Cyan

foreach ($scope in @("Service", "Client")) {
    $regPath = "HKLM:\SOFTWARE\Policies\Microsoft\Windows\WinRM\$scope"
    if (!(Test-Path $regPath)) { New-Item -Path $regPath -Force | Out-Null }
    New-ItemProperty -Path $regPath -Name "AllowBasic" -PropertyType DWord -Value 0 -Force | Out-Null
    Write-Host "WinRM $scope Basic Auth disabled via policy." -ForegroundColor Green
}

Try {
    winrm set winrm/config/service/auth "@{Basic=`"false`"}" | Out-Null
    winrm set winrm/config/client/auth "@{Basic=`"false`"}" | Out-Null
    Write-Host "WinRM config updated." -ForegroundColor Green
} Catch {
    Write-Host "WinRM config update skipped (already enforced by policy)." -ForegroundColor Yellow
}

Write-Host "`nService Auth:" -ForegroundColor Cyan; winrm get winrm/config/service/auth
Write-Host "`nClient Auth:"  -ForegroundColor Cyan; winrm get winrm/config/client/auth
Write-Host "Restart WinRM service if required." -ForegroundColor Green
WN11-AC-000010 Account lockout threshold must be 3 or fewer invalid logon attempts Medium — CAT II

A threshold of 0 disables lockout entirely; values above 3 provide insufficient brute-force protection. Applies via secedit (Local Security Policy). On domain-joined systems, verify the domain GPO is also compliant — it may override local policy.

PowerShell 
<#
.SYNOPSIS  Remediates STIG WN11-AC-000010 — Account lockout threshold (max 3 attempts)
.NOTES     CCI: CCI-000044 | Must run as Administrator | No reboot required
#>

#Requires -RunAsAdministrator

$RequiredThreshold = 3  # Adjust to 1 or 2 for stricter posture

function Get-LockoutThreshold {
    $line = (net accounts 2>&1) | Where-Object { $_ -match 'Lockout threshold' }
    if ($line -match ':\s*(\S+)') {
        $raw = $Matches[1].Trim()
        return ($raw -ieq 'Never') ? 0 : [int]$raw
    }
    return $null
}

function Set-LockoutThreshold([int]$Threshold) {
    $cfg = Join-Path $env:TEMP 'stig_ac000010.inf'
    $db  = Join-Path $env:TEMP 'stig_ac000010.sdb'
    $log = Join-Path $env:TEMP 'stig_ac000010.log'

    @"
[Unicode]
Unicode=yes
[System Access]
LockoutBadCount = $Threshold
[Version]
signature="`$CHICAGO`$"
Revision=1
"@ | Set-Content -Path $cfg -Encoding Unicode

    secedit /configure /db $db /cfg $cfg /log $log /quiet 2>&1 | Out-Null
    Remove-Item $cfg, $db, $log -ErrorAction SilentlyContinue
}

$current = Get-LockoutThreshold
Write-Host "Current threshold: $current"

if ($current -ge 1 -and $current -le $RequiredThreshold) {
    Write-Host "[PASS] Already compliant." -ForegroundColor Green; exit 0
}

Set-LockoutThreshold -Threshold $RequiredThreshold
Start-Sleep -Seconds 2

$verified = Get-LockoutThreshold
if ($verified -ge 1 -and $verified -le $RequiredThreshold) {
    Write-Host "[PASS] Threshold = $verified — compliant." -ForegroundColor Green
} else {
    Write-Host "[FAIL] Threshold = $verified — check domain GPO override." -ForegroundColor Red
    exit 1
}
WN11-CC-000280 Remote Desktop Services must always prompt for passwords upon connection Medium

Without this setting, stored credentials in an RDP connection file can be used to silently authenticate to a session — bypassing authentication entirely. A reboot or gpupdate may be required.

PowerShell 
<#
.SYNOPSIS  Remediates STIG WN11-CC-000280 — RDP must always prompt for password
.NOTES     Date: 2026-05-03 | STIG ID: WN11-CC-000280
#>

Write-Host "Configuring RDP password prompt policy..." -ForegroundColor Cyan

$regPath = "HKLM:\SOFTWARE\Policies\Microsoft\Windows NT\Terminal Services"
if (!(Test-Path $regPath)) { New-Item -Path $regPath -Force | Out-Null }

New-ItemProperty -Path $regPath -Name "fPromptForPassword" -PropertyType DWord -Value 1 -Force | Out-Null

$value = (Get-ItemProperty -Path $regPath -Name "fPromptForPassword" -ErrorAction SilentlyContinue).fPromptForPassword

if ($value -eq 1) {
    Write-Host "[PASS] WN11-CC-000280 — Compliant." -ForegroundColor Green
} else {
    Write-Host "[FAIL] Setting not applied correctly." -ForegroundColor Red
}
WN11-CC-000005 Camera access from the lock screen must be disabled Medium — CAT II

Allowing camera access from the lock screen permits unauthorized video/image capture without system authentication. CCI: CCI-000366. No reboot required; may need gpupdate on domain-joined systems.

PowerShell 
<#
.SYNOPSIS  Remediates STIG WN11-CC-000005 — Disable lock screen camera access
.NOTES     CCI: CCI-000366 | Must run as Administrator
#>

#Requires -RunAsAdministrator

$RegPath     = 'HKLM:\SOFTWARE\Policies\Microsoft\Windows\Personalization'
$ValueName   = 'NoLockScreenCamera'
$RequiredVal = 1

$current = try {
    (Get-ItemProperty -Path $RegPath -Name $ValueName -ErrorAction Stop).$ValueName
} catch { $null }

if ($current -eq $RequiredVal) {
    Write-Host "[PASS] Already compliant — no changes required." -ForegroundColor Green
    exit 0
}

if (-not (Test-Path $RegPath)) { New-Item -Path $RegPath -Force | Out-Null }

New-ItemProperty -Path $RegPath -Name $ValueName -Value $RequiredVal -PropertyType DWord -Force | Out-Null

$verify = (Get-ItemProperty -Path $RegPath -Name $ValueName -ErrorAction SilentlyContinue).$ValueName

if ($verify -eq $RequiredVal) {
    Write-Host "[PASS] WN11-CC-000005 — Compliant. (NoLockScreenCamera = 1)" -ForegroundColor Green
} else {
    Write-Host "[FAIL] Registry value not applied correctly." -ForegroundColor Red
    exit 1
}
WN11-CC-000185 Default autorun behavior must be configured to prevent autorun commands Medium

Autorun commands on removable media are a common malware delivery vector. This script disables autorun across all drive types and fully disables Autoplay via Group Policy registry settings.

PowerShell 
<#
.SYNOPSIS  Remediates STIG WN11-CC-000185 — Disable Autorun commands
.NOTES     Run as Administrator
#>

Write-Host "Applying STIG WN11-CC-000185..." -ForegroundColor Cyan

$Path = "HKLM:\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\Explorer"
if (-not (Test-Path $Path)) { New-Item -Path $Path -Force | Out-Null }

# Disable Autorun for all drive types (0xFF = 255 = all drives)
New-ItemProperty -Path $Path -Name "NoDriveTypeAutoRun" -PropertyType DWord -Value 255 -Force | Out-Null
New-ItemProperty -Path $Path -Name "NoAutorun"          -PropertyType DWord -Value 1   -Force | Out-Null

# Disable Autoplay for non-volume devices via policy
$PolicyPath = "HKLM:\SOFTWARE\Policies\Microsoft\Windows\Explorer"
if (-not (Test-Path $PolicyPath)) { New-Item -Path $PolicyPath -Force | Out-Null }

New-ItemProperty -Path $PolicyPath -Name "NoAutoplayfornonVolume" -PropertyType DWord -Value 1 -Force | Out-Null

$result = (Get-ItemProperty -Path $Path -Name "NoDriveTypeAutoRun" -ErrorAction SilentlyContinue).NoDriveTypeAutoRun

if ($result -eq 255) {
    Write-Host "[PASS] WN11-CC-000185 successfully remediated." -ForegroundColor Green
} else {
    Write-Host "[FAIL] Remediation incomplete." -ForegroundColor Red
}
