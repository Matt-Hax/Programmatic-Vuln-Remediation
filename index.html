STIG WN11-CC-000305

Indexing of encrypted files must be turned off.

<#
.SYNOPSIS
    Remediates STIG WN11-CC-000305 - Disable indexing of encrypted files

.DESCRIPTION
    Disables Windows Search indexing of encrypted files by setting the
    appropriate Group Policy registry value.

.NOTES
    Run as Administrator
#>

Write-Host "Applying STIG WN11-CC-000305 remediation..." -ForegroundColor Cyan

# Registry path for Windows Search policy
$Path = "HKLM:\SOFTWARE\Policies\Microsoft\Windows\Windows Search"

# Create key if it doesn't exist
if (-not (Test-Path $Path)) {
    New-Item -Path $Path -Force | Out-Null
}

# Disable indexing of encrypted files
New-ItemProperty -Path $Path `
    -Name "AllowIndexingEncryptedStoresOrItems" `
    -PropertyType DWord `
    -Value 0 `
    -Force | Out-Null

# Restart Windows Search service to apply change
Write-Host "Restarting Windows Search service..." -ForegroundColor Yellow
Restart-Service -Name "WSearch" -Force -ErrorAction SilentlyContinue

# Validation
$result = Get-ItemProperty -Path $Path -Name "AllowIndexingEncryptedStoresOrItems" -ErrorAction SilentlyContinue

if ($result.AllowIndexingEncryptedStoresOrItems -eq 0) {
    Write-Host "STIG WN11-CC-000305 successfully remediated." -ForegroundColor Green
} else {
    Write-Host "STIG remediation FAILED." -ForegroundColor Red
}

Indexing of encrypted files may expose sensitive data. This setting prevents encrypted files from being indexed.

STIG WN11-CC-000326

PowerShell script block logging must be enabled on Windows 11.

<#

.DESCRIPTION
    Enables PowerShell Script Block Logging to ensure full visibility
    of executed PowerShell commands for auditing and security monitoring.

.NOTES
    Run as Administrator

}

New-ItemProperty -Path $Path `
    -Name "EnableScriptBlockLogging" `
    -PropertyType DWord `
    -Value 1 `
    -Force | Out-Null

# Optional but recommended: enable module logging as well (good for SIEM correlation)
$ModulePath = "HKLM:\SOFTWARE\Policies\Microsoft\Windows\PowerShell\ModuleLogging"
if (-not (Test-Path $ModulePath)) {
    New-Item -Path $ModulePath -Force | Out-Null

    -Name "EnableModuleLogging" `
    -PropertyType DWord `
    -Value 1 `
    -Force | Out-Null

# Ensure module logging includes all modules
$ModuleSettingsPath = "$ModulePath\ModuleNames"
if (-not (Test-Path $ModuleSettingsPath)) {
    New-Item -Path $ModuleSettingsPath -Force | Out-Null
}

New-ItemProperty -Path $ModuleSettingsPath `
    -Name "*" `
    -PropertyType String `
    -Value "*" `
    -Force | Out-Null

# Validation
$check = Get-ItemProperty -Path $Path -Name "EnableScriptBlockLogging" -ErrorAction SilentlyContinue

if ($check.EnableScriptBlockLogging -eq 1) {
    Write-Host "STIG WN11-CC-000326 successfully remediated." -ForegroundColor Green
} else {
    Write-Host "STIG remediation FAILED." -ForegroundColor Red
}

Maintaining an audit trail of system activity logs can help identify configuration errors, troubleshoot service disruptions, and analyze compromises that have occurred, as well as detect attacks. Audit logs are necessary to provide a trail of evidence in case the system or network is compromised. Collecting this data is essential for analyzing the security of information assets and detecting signs of suspicious and unexpected behavior. Enabling PowerShell script block logging will record detailed information from the processing of PowerShell commands and scripts. This can provide additional detail when malware has run on a system.

STIG WN11-00-000160

Disables SMBv1 protocol.

<#
.SYNOPSIS
    Remediates STIG WN11-00-000160 by disabling SMBv1 on Windows 11 systems.

.DESCRIPTION
    The Server Message Block (SMB) v1 protocol is an outdated and insecure protocol.
    This script disables SMBv1 via Windows Optional Features and registry configuration.

.NOTES
    Author: YourName
    Date: 2026-05-03
    STIG ID: WN11-00-000160
#>

Write-Host "Checking SMBv1 status..." -ForegroundColor Cyan

# Disable SMBv1 via Windows Optional Feature (client component)
Try {
    Disable-WindowsOptionalFeature -Online -FeatureName "SMB1Protocol" -NoRestart -ErrorAction Stop
    Write-Host "SMBv1 Windows Feature disabled successfully." -ForegroundColor Green
}
Catch {
    Write-Host "SMBv1 feature may already be disabled or not present." -ForegroundColor Yellow
}

# Disable SMBv1 via registry (additional enforcement)
$registryPath = "HKLM:\SYSTEM\CurrentControlSet\Services\LanmanServer\Parameters"

If (!(Test-Path $registryPath)) {
    New-Item -Path $registryPath -Force | Out-Null
}

New-ItemProperty -Path $registryPath -Name "SMB1" -PropertyType DWORD -Value 0 -Force | Out-Null

Write-Host "Registry settings applied to disable SMBv1." -ForegroundColor Green

# Verify SMBv1 is disabled
Write-Host "`nVerifying SMBv1 state..." -ForegroundColor Cyan
Get-WindowsOptionalFeature -Online -FeatureName "SMB1Protocol"

Write-Host "`nRemediation complete. A reboot may be required for full effect." -ForegroundColor Green

SMBv1 is a legacy protocol that uses the MD5 algorithm as part of SMB. MD5 is known to be vulnerable to a number of attacks such as collision and preimage attacks as well as not being FIPS compliant. Disabling SMBv1 support may prevent access to file or print sharing resources with systems or devices that only support SMBv1. File shares and print services hosted on Windows Server 2003 are an example, however Windows Server 2003 is no longer a supported operating system. Some older Network Attached Storage (NAS) devices may only support SMBv1

STIG WN11-CC-000315

Windows Installer must not always install with elevated privileges

<#
.SYNOPSIS
    Remediates STIG WN11-CC-000315
    Windows Installer must not always install with elevated privileges

.DESCRIPTION
    The Windows Installer "AlwaysInstallElevated" policy allows non-admin users
    to install software with SYSTEM-level privileges. This is a CAT I (High)
    finding and must be disabled in both HKLM and HKCU hives.

.STIG
    ID:       WN11-CC-000315
    Severity: CAT I (High)
    Fix:      Set AlwaysInstallElevated to 0 in both registry hives

.NOTES
    Must be run as Administrator
#>

# ─────────────────────────────────────────
# Requires Administrator
# ─────────────────────────────────────────
If (-NOT ([Security.Principal.WindowsPrincipal] `
    [Security.Principal.WindowsIdentity]::GetCurrent()).IsInRole(`
    [Security.Principal.WindowsBuiltInRole] "Administrator")) {
    Write-Warning "This script must be run as Administrator. Exiting."
    Exit 1
}

# ─────────────────────────────────────────
# Registry Paths
# ─────────────────────────────────────────
$HKLMPath = "HKLM:\SOFTWARE\Policies\Microsoft\Windows\Installer"
$HKCUPath = "HKCU:\SOFTWARE\Policies\Microsoft\Windows\Installer"
$ValueName = "AlwaysInstallElevated"
$RequiredValue = 0

# ─────────────────────────────────────────
# Function: Check Current Value
# ─────────────────────────────────────────
Function Get-STIGStatus {
    param ($Path, $Name)

    If (Test-Path $Path) {
        $val = Get-ItemProperty -Path $Path -Name $Name -ErrorAction SilentlyContinue
        If ($null -ne $val.$Name) {
            Return $val.$Name
        }
    }
    Return "Not Set"
}

# ─────────────────────────────────────────
# Check Before Remediation
# ─────────────────────────────────────────
Write-Host "`n[*] Checking current state before remediation..." -ForegroundColor Cyan

$HKLMBefore = Get-STIGStatus -Path $HKLMPath -Name $ValueName
$HKCUBefore = Get-STIGStatus -Path $HKCUPath -Name $ValueName

Write-Host "  HKLM AlwaysInstallElevated : $HKLMBefore"
Write-Host "  HKCU AlwaysInstallElevated : $HKCUBefore"

# ─────────────────────────────────────────
# Remediate HKLM
# ─────────────────────────────────────────
Write-Host "`n[*] Remediating HKLM..." -ForegroundColor Yellow

Try {
    If (-Not (Test-Path $HKLMPath)) {
        New-Item -Path $HKLMPath -Force | Out-Null
        Write-Host "  [+] Created registry path: $HKLMPath"
    }
    Set-ItemProperty -Path $HKLMPath -Name $ValueName -Value $RequiredValue -Type DWord -Force
    Write-Host "  [+] HKLM\$ValueName set to $RequiredValue" -ForegroundColor Green
} Catch {
    Write-Error "  [-] Failed to remediate HKLM: $_"
}

# ─────────────────────────────────────────
# Remediate HKCU
# ─────────────────────────────────────────
Write-Host "`n[*] Remediating HKCU..." -ForegroundColor Yellow

Try {
    If (-Not (Test-Path $HKCUPath)) {
        New-Item -Path $HKCUPath -Force | Out-Null
        Write-Host "  [+] Created registry path: $HKCUPath"
    }
    Set-ItemProperty -Path $HKCUPath -Name $ValueName -Value $RequiredValue -Type DWord -Force
    Write-Host "  [+] HKCU\$ValueName set to $RequiredValue" -ForegroundColor Green
} Catch {
    Write-Error "  [-] Failed to remediate HKCU: $_"
}

# ─────────────────────────────────────────
# Verify After Remediation
# ─────────────────────────────────────────
Write-Host "`n[*] Verifying remediation..." -ForegroundColor Cyan

$HKLMAfter = Get-STIGStatus -Path $HKLMPath -Name $ValueName
$HKCUAfter = Get-STIGStatus -Path $HKCUPath -Name $ValueName

Write-Host "  HKLM AlwaysInstallElevated : $HKLMAfter"
Write-Host "  HKCU AlwaysInstallElevated : $HKCUAfter"

# ─────────────────────────────────────────
# Pass/Fail Result
# ─────────────────────────────────────────
Write-Host ""
If ($HKLMAfter -eq $RequiredValue -and $HKCUAfter -eq $RequiredValue) {
    Write-Host "  [PASS] WN11-CC-000315 remediation successful." -ForegroundColor Green
} Else {
    Write-Host "  [FAIL] WN11-CC-000315 remediation may not be complete. Review values above." -ForegroundColor Red
}

Write-Host "`n[*] Done. Run 'gpupdate /force' to apply Group Policy changes.`n" -ForegroundColor Cyan

Prevents silent privilege escalation.

STIG WN11-AC-000010

The account lockout threshold must be configured to 3 or fewer invalid logon attempts. 

<#
.SYNOPSIS
    Remediates STIG WN11-AC-000010 - The account lockout threshold must be
    configured to 3 or fewer invalid logon attempts.

.DESCRIPTION
    STIG ID   : WN11-AC-000010
    Severity  : CAT II (Medium)
    CCI       : CCI-000044
    Rule ID   : SV-253298r958388_rule
    Vuln ID   : V-253298

    Finding:
        The account lockout feature prevents brute-force password attacks by
        locking an account after a defined number of consecutive failed logon
        attempts. A threshold of 0 disables lockout entirely, and a value
        greater than 3 makes the protection too weak. The required value is
        between 1 and 3 (inclusive).

    Required setting:
        Policy  : Account lockout threshold
        Value   : 3 or fewer invalid logon attempts (1–3)
        Tool    : Local Security Policy (secedit) / net accounts

    Group Policy equivalent:
        Computer Configuration >> Windows Settings >> Security Settings >>
        Account Policies >> Account Lockout Policy >>
        "Account lockout threshold" = 3 or fewer

.NOTES
    - Must be run as Administrator (elevated PowerShell).
    - This script uses 'net accounts' to read the current threshold and
      'secedit' to apply the policy — the same mechanism as Local Group Policy.
    - On domain-joined systems, the domain Account Lockout Policy will override
      local policy. Verify the domain GPO is also compliant.
    - No reboot is required; the change is effective immediately.
    - The script sets the threshold to 3 (the maximum compliant value). Adjust
      $RequiredThreshold below if your organisation mandates a stricter value.

.EXAMPLE
    .\Remediate-WN11-AC-000010.ps1

.EXAMPLE
    # To enforce a stricter threshold of 2:
    # Edit the line below:  $RequiredThreshold = 2
#>

#Requires -RunAsAdministrator

Set-StrictMode -Version Latest
$ErrorActionPreference = 'Stop'

# ── Configuration ─────────────────────────────────────────────────────────────
$StigId            = 'WN11-AC-000010'
$RequiredThreshold = 3        # STIG maximum; set lower for stricter posture (1–3)
$MinThreshold      = 1        # Must not be 0 (that disables lockout)

# ── Helper: write a timestamped, coloured status line ─────────────────────────
function Write-Status {
    param(
        [string]$Message,
        [ValidateSet('Info','Pass','Fail','Action','Warn')]
        [string]$Level = 'Info'
    )
    $colour = switch ($Level) {
        'Pass'   { 'Green'  }
        'Fail'   { 'Red'    }
        'Action' { 'Cyan'   }
        'Warn'   { 'Yellow' }
        default  { 'White'  }
    }
    $ts = Get-Date -Format 'yyyy-MM-dd HH:mm:ss'
    Write-Host "[$ts] [$StigId] [$Level] $Message" -ForegroundColor $colour
}

# ── Helper: get the current lockout threshold via 'net accounts' ──────────────
function Get-LockoutThreshold {
    $output    = net accounts 2>&1
    $line      = $output | Where-Object { $_ -match 'Lockout threshold' }
    if ($line -match ':\s*(\S+)') {
        $raw = $Matches[1].Trim()
        # 'Never' means 0 (disabled)
        if ($raw -ieq 'Never') { return 0 }
        if ($raw -match '^\d+$') { return [int]$raw }
    }
    return $null
}

# ── Helper: apply the threshold via secedit ───────────────────────────────────
function Set-LockoutThreshold {
    param([int]$Threshold)

    $tmpDir  = $env:TEMP
    $cfgFile = Join-Path $tmpDir 'stig_ac000010.inf'
    $dbFile  = Join-Path $tmpDir 'stig_ac000010.sdb'
    $logFile = Join-Path $tmpDir 'stig_ac000010.log'

    # Build a minimal Security Template (.inf) for secedit
    $infContent = @"
[Unicode]
Unicode=yes
[System Access]
LockoutBadCount = $Threshold
[Version]
signature="`$CHICAGO`$"
Revision=1
"@

    Set-Content -Path $cfgFile -Value $infContent -Encoding Unicode

    Write-Status "Applying policy via secedit (threshold = $Threshold)..." Action

    $result = secedit /configure /db $dbFile /cfg $cfgFile /log $logFile /quiet 2>&1

    # Clean up temp files
    Remove-Item $cfgFile, $dbFile, $logFile -ErrorAction SilentlyContinue

    return $result
}

Write-Status "Starting remediation for $StigId" Info
Write-Status "Control: Account lockout threshold must be 1–$RequiredThreshold invalid logon attempts" Info
Write-Host   ""

# ── Pre-check ─────────────────────────────────────────────────────────────────
Write-Status "PRE-CHECK" Info
$currentThreshold = Get-LockoutThreshold

if ($null -eq $currentThreshold) {
    Write-Status "Could not determine current lockout threshold. Proceeding with remediation." Warn
} elseif ($currentThreshold -eq 0) {
    Write-Status "Account lockout threshold = 0 (DISABLED) -- FINDING" Fail
} elseif ($currentThreshold -gt $RequiredThreshold) {
    Write-Status "Account lockout threshold = $currentThreshold (exceeds maximum of $RequiredThreshold) -- FINDING" Fail
} else {
    Write-Status "Account lockout threshold = $currentThreshold (within compliant range 1–$RequiredThreshold) -- ALREADY COMPLIANT" Pass
    Write-Host   ""
    Write-Status "No changes required. Exiting." Info
    exit 0
}

Write-Host ""

# ── Remediation ───────────────────────────────────────────────────────────────
Write-Status "REMEDIATION" Action
Write-Status "Setting account lockout threshold to $RequiredThreshold via secedit..." Action

try {
    Set-LockoutThreshold -Threshold $RequiredThreshold
    Write-Status "secedit completed successfully." Action
}
catch {
    Write-Status "secedit failed: $_" Fail
    exit 1
}

Write-Host ""

# ── Post-check / Verification ─────────────────────────────────────────────────
Write-Status "POST-CHECK / VERIFICATION" Info

# Allow a moment for the policy engine to apply the change
Start-Sleep -Seconds 2

$verifyThreshold = Get-LockoutThreshold

if ($null -eq $verifyThreshold) {
    Write-Status "VERIFICATION WARNING: Could not read threshold after remediation. Confirm manually via 'net accounts'." Warn
} elseif ($verifyThreshold -eq 0 -or $verifyThreshold -gt $RequiredThreshold) {
    Write-Status "VERIFICATION FAILED: Threshold = $verifyThreshold (non-compliant). Check for overriding domain GPO." Fail
    Write-Status "Run 'gpresult /r' to check which GPO is controlling this setting." Warn
    exit 1
} else {
    Write-Status "VERIFICATION PASSED: Account lockout threshold = $verifyThreshold -- COMPLIANT" Pass
}

Write-Host ""

# ── Summary ───────────────────────────────────────────────────────────────────
Write-Host "─────────────────────────────────────────────────────────────" -ForegroundColor DarkGray
Write-Status "Remediation Summary" Info
Write-Host   "  STIG ID   : $StigId" -ForegroundColor White
Write-Host   "  Setting   : Account lockout threshold" -ForegroundColor White
Write-Host   "  Value Set : $RequiredThreshold invalid logon attempts" -ForegroundColor White
Write-Host   "  Method    : secedit (Local Security Policy)" -ForegroundColor White
Write-Host   "  Status    : COMPLIANT" -ForegroundColor Green
Write-Host "─────────────────────────────────────────────────────────────" -ForegroundColor DarkGray
Write-Host ""
Write-Status "NOTE: On domain-joined systems, verify the domain GPO also enforces" Warn
Write-Status "      this setting. Local policy may be overridden at the next gpupdate." Warn
Write-Host   "      Location: Computer Configuration >> Windows Settings >> Security Settings >>" -ForegroundColor Yellow
Write-Host   "      Account Policies >> Account Lockout Policy >>" -ForegroundColor Yellow
Write-Host   "      'Account lockout threshold' = $RequiredThreshold or fewer" -ForegroundColor Yellow
Write-Host ""
Write-Status "You can also verify with: net accounts | findstr /i 'lockout'" Warn
Write-Host ""
Write-Status "Remediation complete." Pass

The account lockout feature, when enabled, prevents brute-force password attacks on the system. The higher this value is, the less effective the account lockout feature will be in protecting the local system. The number of bad logon attempts must be reasonably small to minimize the possibility of a successful password attack, while allowing for honest errors made during a normal user logon.

STIG WN11-CC-000280

Remote Desktop Services must require Network Level Authentication (NLA)..

<#
.SYNOPSIS
    Remediates STIG WN11-CC-000280 - Remote Desktop Services must require
    Network Level Authentication (NLA).

.DESCRIPTION
    STIG ID   : WN11-CC-000280
    Severity  : CAT II (Medium)
    CCI       : CCI-001453
    Rule ID   : SV-253404r877394_rule
    Vuln ID   : V-253404

    Finding:
        Remote Desktop Services (RDS) connections that do not require Network
        Level Authentication (NLA) are susceptible to man-in-the-middle attacks
        because a full desktop session is established before the user is
        authenticated. NLA forces credential verification before the session is
        created, reducing the attack surface.

    Required registry state:
        Hive  : HKEY_LOCAL_MACHINE
        Path  : SOFTWARE\Policies\Microsoft\Windows NT\Terminal Services
        Name  : UserAuthentication
        Type  : REG_DWORD
        Value : 1  (Enabled / NLA required)

    Group Policy equivalent:
        Computer Configuration >> Administrative Templates >>
        Windows Components >> Remote Desktop Services >>
        Remote Desktop Session Host >> Security >>
        "Require user authentication for remote connections by using
         Network Level Authentication" = Enabled

.NOTES
    - Must be run as Administrator (elevated PowerShell).
    - On domain-joined systems, Group Policy will re-apply this setting on the
      next gpupdate cycle. Deploy via GPO for persistent enforcement.
    - The script performs a pre-check, applies the fix, and then verifies the
      result. No reboot is required; the change takes effect immediately for
      new RDP connections.

.EXAMPLE
    .\Remediate-WN11-CC-000280.ps1
#>

#Requires -RunAsAdministrator

Set-StrictMode -Version Latest
$ErrorActionPreference = 'Stop'

# ── Constants ────────────────────────────────────────────────────────────────
$StigId      = 'WN11-CC-000280'
$RegPath     = 'HKLM:\SOFTWARE\Policies\Microsoft\Windows NT\Terminal Services'
$RegName     = 'UserAuthentication'
$RegType     = 'DWord'
$RequiredVal = 1

# ── Helper: write a timestamped, coloured status line ────────────────────────
function Write-Status {
    param(
        [string]$Message,
        [ValidateSet('Info','Pass','Fail','Action','Warn')]
        [string]$Level = 'Info'
    )
    $colour = switch ($Level) {
        'Pass'   { 'Green'   }
        'Fail'   { 'Red'     }
        'Action' { 'Cyan'    }
        'Warn'   { 'Yellow'  }
        default  { 'White'   }
    }
    $ts = Get-Date -Format 'yyyy-MM-dd HH:mm:ss'
    Write-Host "[$ts] [$StigId] [$Level] $Message" -ForegroundColor $colour
}

# ── Helper: read the current registry value (returns $null if missing) ───────
function Get-CurrentValue {
    try {
        $prop = Get-ItemProperty -Path $RegPath -Name $RegName -ErrorAction Stop
        return $prop.$RegName
    }
    catch {
        return $null
    }
}

# ── Main ─────────────────────────────────────────────────────────────────────
Write-Status "Starting remediation for $StigId" Info
Write-Status "Control: Remote Desktop Services must require Network Level Authentication (NLA)" Info
Write-Host   ""

# ── Pre-check ────────────────────────────────────────────────────────────────
Write-Status "PRE-CHECK" Info
$currentVal = Get-CurrentValue

if ($null -eq $currentVal) {
    Write-Status "Registry value '$RegName' does not exist under '$RegPath' -- FINDING" Fail
} elseif ($currentVal -ne $RequiredVal) {
    Write-Status "Registry value '$RegName' = $currentVal (expected $RequiredVal) -- FINDING" Fail
} else {
    Write-Status "Registry value '$RegName' = $currentVal -- ALREADY COMPLIANT" Pass
    Write-Host   ""
    Write-Status "No changes required. Exiting." Info
    exit 0
}

Write-Host ""

# ── Remediation ──────────────────────────────────────────────────────────────
Write-Status "REMEDIATION" Action
Write-Status "Ensuring registry key path exists: $RegPath" Action

# Create the registry path if it does not exist
if (-not (Test-Path -Path $RegPath)) {
    New-Item -Path $RegPath -Force | Out-Null

# ── Helper: write a timestamped, coloured status line ─────────────────────────
function Write-Status {
    param(
        [string]$Message,

            return $null

    return $null
}

STIG WN11-CC-000185

The default autorun behavior must be configured to prevent autorun commands.

<#
.SYNOPSIS
    Remediates STIG WN11-CC-000185 - Prevent Autorun commands

.DESCRIPTION
    Configures the system to prevent Autorun commands from executing
    on removable and network drives.

.NOTES
    Run as Administrator
#>

Write-Host "Applying STIG WN11-CC-000185 remediation..." -ForegroundColor Cyan

# Registry path for Autoplay/Autorun policies
$Path = "HKLM:\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\Explorer"

# Create key if missing
if (-not (Test-Path $Path)) {
    New-Item -Path $Path -Force | Out-Null
}

# Disable Autorun for all drive types (0xFF = 255)
New-ItemProperty -Path $Path `
    -Name "NoDriveTypeAutoRun" `
    -PropertyType DWord `
    -Value 255 `
    -Force | Out-Null

# Ensure Autorun is fully disabled
New-ItemProperty -Path $Path `
    -Name "NoAutorun" `
    -PropertyType DWord `
    -Value 1 `
    -Force | Out-Null

# (Optional but recommended) Turn off Autoplay completely
$PolicyPath = "HKLM:\SOFTWARE\Policies\Microsoft\Windows\Explorer"

if (-not (Test-Path $PolicyPath)) {
    New-Item -Path $PolicyPath -Force | Out-Null
}

New-ItemProperty -Path $PolicyPath `
    -Name "NoAutoplayfornonVolume" `
    -PropertyType DWord `
    -Value 1 `
    -Force | Out-Null

# Validation
$result = Get-ItemProperty -Path $Path -Name "NoDriveTypeAutoRun" -ErrorAction SilentlyContinue

if ($result.NoDriveTypeAutoRun -eq 255) {
    Write-Host "STIG WN11-CC-000185 successfully remediated." -ForegroundColor Green
} else {
    Write-Host "STIG remediation FAILED." -ForegroundColor Red
}

Allowing autorun commands to execute may introduce malicious code to a system. Configuring this setting prevents autorun commands from executing.
